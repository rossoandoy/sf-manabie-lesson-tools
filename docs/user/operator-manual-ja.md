# 操作マニュアル（教室長 / 本部事務）

Sandbox E2E Sign-off チェックリスト: [docs/e2e-sandbox-signoff.md](../e2e-sandbox-signoff.md)

## 1. インストール

初回セットアップは **[利用開始ガイド](./getting-started-ja.md)** を参照してください。  
IT 担当者向けの配布手順は **[配布ガイド](./distribution-guide-ja.md)** です。

1. Chrome で拡張 `Manabie Lesson Tools` を読み込む（ZIP 解凍 → `chrome://extensions` → 開発者モード）
2. trg2--extuat（Sandbox）にログイン
3. 拡張 Popup → **ダッシュボードを開く**

## 2. 前提マスタ同期

1. ダッシュボード上部の **前提マスタ同期** をクリック
2. 拠点・年度・拠点コース・クラス・教室・講師が取得されることを確認（エラー toast が出ないこと）
3. コマ組 **表示設定** の **ブース数** が Account の値と一致すること（trg2--extuat では `TRG_BoothCount__c`。手動変更不可）
4. 「Manabie登録」タブで件数を確認

## 3. コマ組での授業入力（Phase 12 以降）

授業スケジュールタブは廃止され、日常入力は **コマ組** に集約されています。

1. **コマ組** タブで教室名・拠点（Account）・ブース数・時限を設定
2. **ブース表** モードで日付×ブース×時限に講師・生徒を入力
3. **カレンダー** モードで授業スケジュール（Lesson）を編集 — コンテキストバー「リスト / タイムスロット」
4. **Manabie登録** タブで ImportPlan を確認 → **Manabieへ登録**（確認フレーズ `EXECUTE SANDBOX`）

CSV ダウンロードは任意のローカルバックアップ用途のみ（通常運用では API 送信を使用）。

## 4. 休校日（Phase 1 / Phase 18）

1. **休校日** タブで日付をクリック（**緑枠** = 選択中、凡例を参照）
2. **表示する休校日名** と年度を入力
3. エクスポート（任意）で CSV 出力可能

## 5. Manabie登録（Phase 18 — Lesson_Slot 主役）

1. **Manabie登録** タブ上部で **Lesson_Slot__c** プレビューを確認 → **授業データ送信**
2. 旧「授業スケジュール Import」は折りたたみ **レガシー** セクション内
3. 休校日 Manabie 登録はタブ下部（Phase 1.5）

## 6. トラブルシューティング

| 症状 | 対処 |
|------|------|
| セッションなし | Salesforce を再ログイン |
| Manabieへ登録が無効 | 前提マスタ同期 + 授業入力 + Manabie登録タブで error 解消 |
| Production org | Sandbox のみ Execute 可能 |

## 7. ロールバック

Sandbox で作成されたレコードは Salesforce 標準 UI または Data Loader で削除。Execution ログの Salesforce ID を参照。

## 8. コマ組（Phase 2A / Phase 10–13 レイアウト）

1. **コマ組** タブで教室名・拠点（Account）・ブース数・時限を設定
   - **ブース数**は Account から自動（変更不可）。前提マスタ同期後に反映
   - **Phase 10 / 18**: Affiliation は Account Id のみ取得 → マスタ locations または Account 直接 SOQL で解決（relationship FLS 回避）
   - 自動設定に失敗した場合、セットアップ checklist に SF エラー詳細が表示されます
   - 手動が必要な場合: checklist「所属校舎を確認」→ 拠点 (Account) を選択（「手動で変更」トグル）
2. コンテキストバー: **カレンダー / ブース表** 切替、週ナビ + **表示週レンジ**。折りたたみは左 **表示設定** / 右 **コマ組操作** 見出し横
3. ブース表は Excel ブース表と同型: **日付｜ブース｜各時限×（講師｜生徒｜学年｜教科）** — 週全体が1テーブル、横スクロール時 **日付/ブース列は固定（sticky）**
4. 生徒/講師/教科: **▾ ボタン**（点線枠）をクリック → 検索モーダルで選択。入力欄への直接入力ではありません
5. **学年** は生徒選択時に自動入力（編集不可）
6. 右パネル: **選択中コマ** / **登録済み繰り返し**（管理のみ） / **授業一覧プレビュー**
7. 左設定パネル: **一括削除（F04）** — 生徒 or 講師名を選び、表示週の該当コマを削除（preview + confirm）
8. 席1に **1:1 契約**（`MANAERP__Lesson_Capacity__c`）の生徒を入れると席2が `(1:1枠)` でグレーアウト
9. 休校日は自動で入力不可（休講化）
10. **出欠記録**（右パネル）: 席1/席2 を切替。出席=青・欠席=赤・振替=黒（GAS 同配色）。ブース表の該当席行にも同色反映。SF への反映は **授業一覧 Sync Dock「Manabie 出欠同期（3B）」**
11. **授業タイムスロット** は **授業一覧** タブのツールバーから side drawer を開く
12. Lesson 未生成日がある場合、ツールバー下に **Schedule Gap 警告** が表示されます

### 翌年度準備（R04 / Phase 14）

1. 左設定パネル **翌年度を準備（R04）** でプレビュー（削除対象年度・コマ数・振替保護件数）を確認
2. 確認トークン `FY{前年度}` を入力（例: 2026 年度準備 → `FY2025`）
3. 実行前に **JSON バックアップ** が自動ダウンロードされる
4. 前年度（currentYear−1）の cells / slotMeta を削除（**振替元/先あり行は保護**）
5. **生徒/講師定期** を自動整理 — 前年度のみの repeat は `ended`、新年度跨ぎは `startDate` を新年度 4/1 に更新
6. 年度が +1 され、授業一覧の期間デフォルトも新年度 4/1〜3/31 に更新

詳細: [phase14-r04-spike.md](../phase14-r04-spike.md)

## 9. 授業一覧 / Manabie 同期（Sync Dock / Phase 4–13）

ダッシュボードのタブ順は **[コマ組] [授業一覧] [回数報告] …** です。日常の SF 同期は **授業一覧 下部の Sync Dock** に集約されています。

1. **授業一覧** タブで Excel 相当列（講師・学年・種別・振替・備考）を編集
2. 絞り込み: 生徒 or 講師 + **名前検索モーダル**、期間（自/至）、**振替待ちのみ**（出欠=振替 かつ 振替先未入力）、**未同期のみ**
3. 200行超の年度範囲でも tbody 仮想スクロールで滑らかに表示
4. 下部 **Sync Dock** の **授業データ送信（F19）** — モーダルで `EXECUTE SANDBOX` を入力
5. **Manabie 出欠同期（3B）** — 同上（出席/欠席/休講。振替・未確定はスキップ）
6. Schedule Gap 警告が出た場合、Manabie 側で Lesson 未生成 — 3B は該当日をスキップ
7. **Manabie Session 作成（3B+）** — Lesson あり / Session なし / 出席・欠席・休講。作成後に 3B 出欠同期を実行
8. **Manabie 振替登録（3C）** — 振替行（transferFrom あり）→ Reallocation create。振替元 Session + 振替先 Lesson が必要
9. 成功時はトースト + 授業一覧 **SF 列**の 3 分割ドット（F19|3B|3C）で同期状態を確認（ホバーで詳細）

### Manabie データ更新（Phase 8）

コマ組 / 授業一覧 を編集しても **自動では Manabie を再取得しません**（API 節約）。

1. 編集後、Sync Dock 上部が **「Manabie データ要更新」**（amber）になる
2. **Manabie データ更新** ボタンをクリック → Lesson / Session / Schedule Gap を再取得
3. 3B / 3C / F19 を Execute する前に更新推奨（未更新のまま Execute すると確認ダイアログ）
4. 授業一覧ツールバー **「未同期のみ」** で SF 未反映行を絞り込み
5. SF 列: 緑ドット = 同期済、点線ボーダー / 薄グレー行 = 編集後要再同期

### 振替の Manabie 同期フロー（Phase 7）

振替は **3B 出欠同期では書き込まれません**（`attendance=振替` はスキップ）。Manabie へ反映する手順:

1. コマ組で振替を登録すると授業一覧に `振替` + `transferFrom`（元日）/ `transferTo`（先日）が付きます
2. **振替元** — 元日の Student Session が Manabie に存在すること（3B で出欠済み、または既存 Session）
3. **振替先 Lesson** — 先日の `MANAERP__Lesson__c` が生成済みであること（未生成週は Schedule Gap 警告）
4. （任意）**振替先 Session** — 先日に Session が無い場合、**Manabie Session 作成（3B+）** で先日分を create → **3B** で出欠を反映
5. Sync Dock **Manabie 振替登録（3C）** を Execute — `MANAERP__Reallocation__c` を 1 件 create（Status: Open）
6. 先日 Session が無くても 3C は create 可能（Phase 7 スコープ）。先日 Session の自動 create + Reallocation 一括は Phase 7 外

詳細: [phase7-reallocation-spike.md](../phase7-reallocation-spike.md)

## 10. 回数報告（Phase 2E–2G / 3A / 3B）

1. **回数報告** タブで生徒・年度を選択
2. **データソース** を「コマ組」または「Manabie SF」に切替
3. **回数報告を更新** で右表（予定/実施）を集計
4. **請求データ同期（F13）** で左表「請求中/支払済」コマ数を充填（請求中: 全 bill item の `TRG_Purchased_Slot__c` 合計。支払済: 入金済行のみ — [phase6-paidkoma-spike.md](../phase6-paidkoma-spike.md)）
5. Manabie SF 選択時、月別 executed 不一致行が黄色ハイライトされます
6. F13 未同期時は **Sync Dock へ** リンクから授業一覧にジャンプできます
7. **印刷（A4）** / **CSV 出力（F11）** — 保護者署名・日付は **印刷時のみ** 表示
8. **請求対象月** は年度内プルダウンから選択（同期済み月にラベル表示）

## 11. セットアップチェックリスト（Phase 4）

初回または未設定時、ヘッダー下にチェックリストが表示されます。

1. **前提マスタ同期**（ヘッダーボタン）
2. **コマ組** で拠点（Account）を選択
3. （任意）**回数報告** で請求データ F13 を初回同期

## 12. コマ組 仮想スクロール（Phase 5 / Phase 15 復元）

ブース数 × 時限 × 曜日 × 2席 のセル数が閾値（**400**）を超えると、コマ組グリッドは **2 日分ずつ** 表示し、ツールバーの **◀ 日 / 日 ▶** でスクロールします。表示中の日付レンジもツールバーに表示されます。週ナビ（◀ 週 / 週 ▶ / Today）では表示位置が先頭日にリセットされます（`dayScrollOffset` は session に永続化）。

## 13. 休校日とコマ組の連動（Phase 2H）

休校日を追加すると該当日のコマは **休講** になります。休校日を削除すると、休講化前の出欠に **自動復元** されます（手動で休講にしたコマは対象外）。

## 14. コマ組 UX（Phase 9 / v0.3.0）

### コマ操作（F04）

1. コマをクリックして選択
2. 右サイドバー: **コピー** → 先コマで **貼付**、または **移動** → 移動先コマをクリック
3. 日見出し: **一括出席** / **休校化** / **全コマ削除**

### 生徒繰り返し（F07）

1. コマ組で配置元コマを選択（任意 — 選択中コマから曜日/時限/ブース/席/生徒名/教科が prefill）
2. 右パネル **繰り返し配置** → **生徒** タブ
3. **検索** で生徒名を catalog から選択 → 教科・間隔・開始/終了日を確認
4. **プレビュー** → skip（休校日/満席）を確認 → **適用**
5. **登録済み繰り返し** から **再配置** または **終了**（終了時も既存コマは削除されません）

### 講師繰り返し（F07）

1. コマ組で配置元コマを選択（任意 — 講師名/曜日/時限/ブースが prefill）
2. 右パネル **繰り返し配置** → **講師** タブ
3. **検索** で講師名を選択 → **プレビュー**（講師衝突日は skip 表示）→ **適用**
4. **終了** で定期を `ended` に（slotMeta は削除されません）

### 学年自動（R06）

前提マスタ同期後、コマ組で生徒名を datalist から選択すると catalog の学年が自動入力されます。

### 振替（Phase 15 振替 v2）

- **振替ウィザード**（コマ組「繰り返し配置」内）: **選択中コマ**から元日/ブース/時限/席を prefill。生徒名は検索モーダルで選択
- **同コマ2席一括**: 席1+席2 両方に生徒がいる場合、チェックで一括振替（`registerTransferPair`）
- **振替待ちリスト**: 項目クリックで該当コマをグリッド上で選択 + スクロール
- **振替待ち** フィルタ: 授業一覧「振替待ちのみ」（出欠=振替 かつ 振替先未入力）
- Manabie 反映: Phase 7 の 3C フロー（[§9](#9-授業一覧--manabie-同期sync-dock--phase-4-13)）

### キーボード（Phase 15）

| 操作 | 動作 |
|------|------|
| 矢印 / Alt+矢印 | コマ移動（時限 / ブース / 日付）— フォーカス field 維持 |
| Tab / Shift+Tab | 講師 → 席1生徒 → 学年 → 教科 → 席2… |
| Ctrl+C / Ctrl+V / Ctrl+X | コピー / 貼付 / 移動モード |
| Escape | クリップボード / 移動キャンセル |

### 入力（Phase 15）

生徒名・講師名のタイプ中は **グリッド全再描画を行わない** ため、フォーカスとカーソル位置が維持されます。

### 週コピー

コマ組ツールバー **前週→今週コピー** — 講師+生徒が揃ったコマのみ、空き先へコピー

### SF 列ドット（Phase 8）

授業一覧 **SF** 列は 3 分割ドット（F19 | 3B | 3C）。緑=同期済、点線/薄グレー=編集後要再同期。詳細はホバー。
